# 应用层代码结构与注释整理设计

## 目标

提高智能车应用层源码的可读性和接口一致性，为后续前瞻转向功能提供清晰的代码基础。本次只整理结构、声明和注释，不改变任何图像、舵机、电机、PID、按键或中断运行行为。

## 修改范围

- `code/config.h`
- `code/image.c`、`code/image.h`
- `code/motor.c`、`code/motor.h`
- `code/pid.c`、`code/pid.h`
- `code/steer.c`、`code/steer.h`
- `user/cpu0_main.c`
- `user/isr.c`

厂商库、驱动层、摄像头驱动和链接配置不修改。

## 头文件结构

每个应用头文件统一使用以下顺序：

1. 文件职责说明。
2. 头文件保护宏。
3. 依赖头文件。
4. 硬件引脚或常量定义。
5. 数据类型定义。
6. 共享变量声明。
7. 公开函数声明。

`motor.h` 的声明与 `motor.c` 对齐：声明实际存在的 `MotorL_SetSpeed()` 和 `MotorR_SetSpeed()`，移除不存在且没有调用者的 `MotorL_Init()`、`Motor_SetSpeed()`。

## 源文件结构

各模块按职责分区：

- 模块全局状态。
- 初始化函数。
- 数据采集或图像预处理。
- 核心控制逻辑。
- 底层输出和通用限制函数。

不进行函数拆分、函数重命名、算法改写或变量类型调整。

## 注释标准

公开函数使用完整中文块注释，至少说明：

- 函数用途。
- 参数含义和单位。
- 返回值。
- 调用位置或周期。
- 与中断、共享变量、硬件输出有关的注意事项。

关键算法内部补充“为什么这样做”的说明，包括 Otsu 阈值、二值化、起点搜索、逐行边线跟踪、丢线保持、中线加权、动态舵机 PID、电机前馈和速度闭环。

中断文件说明每个已启用中断的职责；未使用的空中断仅标记为预留，不填充无意义说明。

## 行为保持约束

- 配置参数数值不变。
- 所有控制公式不变。
- 中断周期和调用顺序不变。
- 图像显示按键逻辑不变。
- 二值化和搜线循环边界不变。
- 电机方向、电机 PWM 和舵机 PWM 输出不变。

## 验证

- 整理前先记录公开函数和配置参数快照。
- 整理后比较函数集合、参数值、控制表达式和中断调用顺序。
- 检查头文件声明与源码定义一致。
- 检查括号平衡、头文件保护宏、重复声明和临时文件。
- 若 TASKING 构建工具可用则执行 Debug 构建；否则只报告静态验证结果，并要求在 ADS 中编译。

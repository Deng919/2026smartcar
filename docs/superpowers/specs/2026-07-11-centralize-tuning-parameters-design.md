# 智能车调参参数集中化设计

## 目标

在不改变小车运动控制行为、控制调用顺序、硬件引脚和运行时数据结构的前提下，将当前分散在图像处理、舵机、PID、电机和主循环中的可调参数集中到一个配置头文件，便于后续调参和维护。

## 范围

### 纳入统一配置的内容

- 图像处理：灰度级数量、极点搜索行、上下搜索范围、左右边线搜索步长、图像阈值偏移、中线位置。
- 中线加权：各图像行的中线权重数组。
- 舵机：舵机 PWM 中值、允许偏转范围。
- 舵机控制：基础 Kp、动态 Kp 增益、Kd、输出低通系数。
- 电机控制：左右电机 PID 的 Kp/Ki/Kd、积分限制、PWM 输出限制、默认目标速度和速度差限制。
- 控制周期：图像/舵机控制周期、编码器/按键/启动控制周期。

### 明确不纳入的内容

- 电机、舵机、编码器的硬件引脚映射。
- 中断优先级、服务核心和底层驱动配置。
- `motor_l`、`motor_r`、PID 状态、图像缓存等运行时状态。
- 控制算法、计算顺序、限幅逻辑和启动逻辑。

## 文件结构

新增：

- `code/config.h`：唯一的调参入口，按“图像、权重、舵机、PID、电机、周期”分区，保留当前参数值。

修改：

- `code/image.h`：删除图像调参宏，改为引用 `config.h`。
- `code/image.c`：删除重复的图像调参宏，使用配置中的参数和中线权重。
- `code/steer.h`：删除舵机调参宏，保留舵机硬件接口声明。
- `code/pid.h`：删除积分限制宏，引用配置文件。
- `code/pid.c`：将 PID 初始值和动态调参常量改为配置值。
- `code/motor.h`：删除 PWM 限制宏，保留电机硬件映射。
- `code/motor.c`：将目标速度、速度差限制和 PID 输出限制改为配置值。
- `user/cpu0_main.c`：将两个 PIT 周期改为配置值。

## 兼容策略

配置项采用预处理宏，保持现有表达式的数值类型和展开方式，避免引入运行时结构体、额外函数调用或新的内存访问。原有控制代码只进行符号替换和必要的头文件引用调整，不修复既有算法问题，也不顺手修改运动控制逻辑。

中线权重作为调参数据集中放在 `config.h` 中，通过配置宏提供初始化列表；图像模块仍拥有自己的运行时访问对象，避免在多个编译单元中重复定义全局变量。

## 验证

由于当前环境没有可用的 TC264/TASKING 编译器入口，验证分三层：

1. 静态检查：确认所有原有调参宏和数值只在 `config.h` 中定义，硬件引脚仍只在原模块中定义。
2. 预处理/引用检查：确认所有使用点都能找到配置符号，头文件没有循环依赖或重复定义。
3. 工程构建：若项目现有 ADS/TASKING 构建命令可用，执行一次完整构建；否则明确记录未执行硬件工具链构建，不宣称编译通过。

验收标准是：配置文件中的初始值与重构前一致；运动控制相关 `.c` 文件的算法语句、调用顺序和硬件映射不发生行为性修改；所有调参项均能从 `config.h` 找到。

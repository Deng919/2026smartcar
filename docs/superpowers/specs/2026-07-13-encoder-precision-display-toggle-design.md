# 编码器精度与图像显示开关设计

## 目标

提高编码器滤波速度的数值精度，并允许使用 KEY_2 在运行中关闭或恢复 TFT 图像刷新，从而降低显示带来的图像处理延迟。

## 编码器速度

- `motorl.encoder_speed` 从 `int` 改为 `float`。
- `encoder_raw` 和 `total_encoder` 保持整数。
- 一阶滤波使用 `0.2f` 和 `0.8f`，保留小数结果。
- 电机 PID 直接读取浮点速度。

## 图像显示开关

- KEY_1 继续控制小车启停，KEY_2 短按切换显示状态。
- 默认开启显示。
- 关闭显示只停止 TFT 绘制，摄像头采集、二值化、搜线、中线计算和控制环继续运行。
- 显示关闭后由 CPU0 主循环清屏一次，禁止在 5ms 电机中断中执行慢速 TFT 操作。
- 再次开启后，从下一帧开始恢复边线、数字和灰度图显示。
- 显示状态由中断写入、主循环读取，因此声明为 `volatile uint8`。

## 验证

- 源码检查确认编码器速度类型和滤波常量为浮点。
- 源码检查确认 KEY_2 切换显示标志，显示关闭分支只清屏一次。
- 检查括号、临时文件和参数引用；当前环境无 TASKING 编译器，因此最终仍需在 ADS 中编译验证。

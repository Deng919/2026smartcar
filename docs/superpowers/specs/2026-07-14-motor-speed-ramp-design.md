# 电机目标速度斜坡设计

## 目标

在不改变图像识别、舵机控制、速度 PID 和现有弯道分档逻辑的前提下，为电机基础目标速度增加斜坡限制，降低起步和直弯速度切换时的突变。

## 行为

- 车辆进入运行状态后，斜坡速度从 0 开始接近速度调度给出的目标值。
- 加速时，每个电机控制周期最多增加 `CONFIG_MOTOR_ACCEL_STEP`，默认值为 2。
- 减速时，每个电机控制周期最多减少 `CONFIG_MOTOR_DECEL_STEP`，默认值为 4。
- 当剩余差值小于对应步长时，直接钳位到目标速度，禁止越过目标。
- KEY_1 切换到停车状态时立即调用 `Motor_Control(0, 0)`，同时清零斜坡状态；停车不经过缓慢减速，以保证安全。
- 再次启动时从 0 重新爬升，不保留上次运行速度。
- 保留用户当前尚未提交的直道速度 130 和中弯速度 110，不修改其他调车参数。

## 代码结构

在 `motor.c` 中增加一个仅负责基础目标速度变化的纯函数和一个当前斜坡速度状态。纯函数不读取图像、不输出 PWM，只根据当前值、目标值和配置步长返回下一周期值。

`car_start()` 继续先根据中线误差选择直道、中弯或急弯目标速度，再把该目标交给斜坡函数，最后调用现有 `Final_Motor_Control()` 生成左右轮差速。这样速度斜坡位于弯道调度与左右轮差速之间，不改变原有控制链路。

## 参数

在 `config.h` 的电机参数区增加：

```c
#define CONFIG_MOTOR_ACCEL_STEP           2
#define CONFIG_MOTOR_DECEL_STEP           4
```

以当前 5 ms 电机控制周期计算，从 0 增加到 130 约需 0.325 秒；从 130 降到 80 约需 0.063 秒。

## 测试

使用脱离硬件的 GCC 合成测试验证生产斜坡函数：

- 从 0 向 10 加速时依次得到 2、4、6、8、10。
- 从 10 向 0 减速时依次得到 6、2、0，不产生负向越界。
- 当前值与目标值相同时保持不变。
- 剩余差值小于步长时直接钳位到目标。
- 源码断言确认停车分支同时清零斜坡状态并立即输出 0。

## 不在本次范围内

- 不修改图像搜线、白色围栏过滤或前瞻算法。
- 不修改舵机 PID、电机 PID、前馈系数和左右轮差速公式。
- 不修改 KEY_1 的奇偶启停语义。
- 保留用户已有的直道速度 130 和中弯速度 110，不回退这两项调车参数。

# 2026 智能车 TC264 工程

基于逐飞科技 TC264 开源库和 Infineon iLLD 的智能车控制工程，使用 MT9V03X 灰度摄像头完成赛道识别，并通过舵机转向、左右轮差速、速度前馈和增量式 PID 实现车辆控制。

项目仍处于实车调试阶段。图像、速度和转向参数集中放在 `code/config.h`，便于针对赛道、光照和机械结构统一调整。

## 开发环境

- 主控：Infineon AURIX TC264D
- IDE：AURIX Development Studio 1.10.10
- 编译器：TASKING C/C++
- 摄像头：MT9V03X 灰度摄像头
- 显示：TFT180
- 执行机构：舵机、左右驱动电机和双编码器
- 基础软件：逐飞科技 TC264 开源库、Infineon iLLD

## 当前功能

- 摄像头 DMA 完整帧同步，避免处理仍在写入的图像。
- Otsu 自动阈值与二值化。
- 左右边线起点搜索和逐行边线跟踪。
- 单行丢线时保持上一行边线，减小中线突变。
- 多行加权中线与一阶滤波。
- 动态舵机 PD 转向控制。
- 左右轮差速、弯道降速、速度前馈和增量式 PID 速度闭环。
- KEY_2 独立关闭 TFT 图像显示，关闭显示后识别和控制仍持续运行。

## 图像识别流程

CPU0 主循环只在 `mt9v03x_finish_flag` 置位后处理一帧图像，当前调用顺序如下：

1. 将 DMA 完成的 `mt9v03x_image` 复制到 `base_image`。
2. 使用 `Ostu()` 计算当前帧灰度阈值。
3. 使用 `set_image_twovalues()` 生成二值图。
4. 使用 `find_jidian()` 在固定搜索行寻找左右边线起点。
5. 使用 `image_deal()` 从近端向远端逐行跟踪左右边线。
6. 当前行找不到边线时，保留上一行边线位置，避免中线突然回到图像中心。
7. 使用 `find_mid_line_weight()` 对各行中线加权并滤波，得到 `final_mid_line`。
8. 周期中断读取最新中线，分别执行舵机转向和电机速度控制。

TFT 绘图不参与识别计算。关闭显示只会跳过耗时的绘图操作，不会停止摄像头处理。

## 转向与速度控制

### 舵机控制

CCU60 CH0 按 `CONFIG_SERVO_CONTROL_PERIOD_MS` 周期运行：

1. `dynamic_pid_value_set()` 根据中线绝对偏差提高舵机比例系数。
2. `servo()` 计算位置式 PD 输出。
3. 输出在舵机中位值上下限幅，防止超过机械转角。

### 电机控制

CCU60 CH1 按 `CONFIG_MOTOR_CONTROL_PERIOD_MS` 周期运行：

1. 读取并滤波左右编码器速度。
2. 扫描按键并处理车辆启停状态。
3. 根据中线绝对偏差选择目标速度档位。
4. 根据中线方向生成左右轮差速目标。
5. 速度前馈提供基础 PWM，增量式 PID 修正实际速度误差。

## 参数调整

所有主要调试参数位于 [`code/config.h`](code/config.h)：

- 图像参数：灰度级、起点搜索行、搜线起止行、横向搜索范围、图像中心和阈值偏移。
- 舵机参数：PWM 中位与范围、输出限幅、基础比例系数、动态比例增益、微分系数和低通系数。
- 电机参数：速度环 PID、积分与 PWM 限幅、目标速度、左右轮速度差、弯道降速阈值和前馈增益。
- 调度参数：舵机控制周期和电机控制周期。
- 中线参数：不同图像行的加权数组。

建议一次只调整一类参数，并记录直道、缓弯、急弯和丢线场景下的变化。提升目标速度前，应先确认边线稳定、舵机无明显振荡、编码器方向正确且电机 PWM 未长期饱和。

## ADS 编译与下载

1. 克隆本仓库。
2. 打开 AURIX Development Studio。
3. 选择 `File -> Import -> Existing Projects into Workspace`。
4. 选择仓库根目录并导入工程。
5. 选择 `Debug` 构建配置，执行 `Incremental Build`。
6. 确认编译无错误后，使用项目调试配置或本机调试器配置下载程序。
7. 首次上车前抬起驱动轮，检查左右编码器符号、电机方向、舵机中位和按键停机功能。

> 应用层部分 C/H 文件使用 GBK/CP936 编码。若中文注释显示异常，请将编辑器文件编码切换为 GBK，不要直接批量转换后提交。

## 实车识别注意事项

- 赛道围栏为白色，围栏与赛道之间存在黑色间隔，围栏可能被误识别为赛道边线。
- 摄像头俯角、曝光、赛道反光和环境光都会影响 Otsu 阈值及黑白跳变位置。
- 搜线结果应同时观察左右边线连续性、赛道宽度和中线变化，不能只根据单帧中线调参。
- 提高车速前应在低速下分别验证直道、连续弯道、急弯和短时丢线。
